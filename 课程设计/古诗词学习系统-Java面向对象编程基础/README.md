# 古诗学习系统

一个基于 Java 控制台的古诗学习系统，适合作为 Java 程序设计课程设计或面向对象编程练习项目。系统围绕古诗数据的读取、查询、学习记录和练习测试展开，实现了较完整的基础业务流程。

## 项目简介

本项目通过文本文件保存古诗数据，使用 Java 集合管理古诗、诗人和学习记录，并通过控制台菜单提供交互式操作。用户可以查询古诗、查看诗人作品、标记已掌握古诗，并基于已掌握内容进行练习测试。

项目重点内容包括：

- Java 面向对象设计
- 集合类 `Map`、`List` 的使用
- 文件读取与对象序列化保存
- 控制台菜单交互
- 简单业务模块划分
- 基础测试类编写

## 功能模块

### 1. 古诗数据管理

- 从 `poems.txt` 文件读取古诗数据
- 解析古诗标题、作者、朝代和诗句内容
- 使用集合保存古诗与诗人信息
- 支持根据标题和诗人进行查询

### 2. 学习模式

- 按古诗标题查询古诗
- 按诗人姓名查询其作品
- 查看古诗详细内容
- 将古诗加入“已掌握”列表
- 查看和移除已掌握古诗
- 查询失败时提供相似标题或诗人提示

### 3. 练习模式

- 标题与诗人练习：根据诗句填写古诗标题和作者
- 诗句填空练习：根据部分诗句补全剩余内容
- 自动统计答题正确数和得分
- 保存练习结果
- 查看历史练习记录

### 4. 学习记录

- 记录已掌握古诗
- 记录每次练习成绩
- 统计平均成绩
- 显示最近练习情况
- 使用对象序列化保存到 `study_record.dat`

## 运行环境

- JDK 8 或以上版本
- IntelliJ IDEA、Eclipse 或其他 Java IDE
- Windows / macOS / Linux 均可运行，但需要根据实际路径调整数据文件路径

## 运行方式

### 方式一：使用 IntelliJ IDEA

1. 打开 IntelliJ IDEA
2. 选择 `Open`，打开项目文件夹
3. 确认 `src` 目录为源码目录
4. 打开 `Main.java`
5. 运行 `main` 方法

### 方式二：使用命令行

进入项目目录后执行：

```bash
javac src/*.java
java -cp src Main
```

如果程序无法读取 `poems.txt`，请检查 `Main.java` 和 `PoemSystem.java` 中的数据文件路径，并根据自己本机项目位置进行修改。

## 数据格式

`poems.txt` 中的古诗数据格式如下：

```text
春晓 - 孟浩然(唐)
春眠不觉晓，
处处闻啼鸟。
夜来风雨声，
花落知多少。
```

每首诗之间使用空行分隔，第一行包含诗名、作者和朝代，后续行为诗句内容。

## 核心类说明

| 类名 | 作用 |
| --- | --- |
| `Main` | 程序入口，负责主菜单显示和功能调度 |
| `PoemSystem` | 系统核心类，负责古诗加载、查询、学习记录管理 |
| `Poem` | 古诗实体类，保存标题、诗人、诗句等信息 |
| `Poet` | 诗人实体类，保存诗人姓名和朝代 |
| `StudyMode` | 学习模式，提供查询和掌握列表管理功能 |
| `ExerciseMode` | 练习模式，提供答题、评分和记录功能 |
| `StudyRecord` | 学习记录类，保存已掌握古诗和练习结果 |

## 项目特点

- 结构清晰，适合于 Java 初学者阅读和扩展
- 使用面向对象方式划分实体类和业务类
- 覆盖文件读取、集合管理、控制台交互、对象序列化等基础知识点
- 功能完整度适合作为小型课程设计项目

## 当前不足

- 控制台界面较简单，没有图形化界面
- 数据文件路径存在硬编码问题，跨设备运行时需要手动调整
- 没有使用数据库，数据量较大时扩展性有限
- 测试类较基础，尚未引入 JUnit 等标准测试框架

## 后续改进方向

- 增加模糊搜索、分类筛选和朝代筛选
